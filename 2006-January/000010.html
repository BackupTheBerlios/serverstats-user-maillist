All Strings in config/graph.php are changeable by the user, they are
part of the configuration. This includes the configuration which graphs
are generated and their names.
So this part of the english-strings is not intended to be translated by
the script, like the titles of the graphs in the includes
config/graph.php (which is an example).

Perhaps it would be an idea to include translated
config/graph.php-files. That could look like this:
config/graph.php -> english
config/graph.php.de_DE -> german
config/graph.php.fr_FR -> french
But this would mean we have to sync all files to have the same
configuration but different titles. Perhaps this should soonest happen
when Serverstats 1.0 is released. Alternative we can update the
documentation to tell the user to change this.
