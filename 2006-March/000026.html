Hi!

I use serverstats at my server, with the traffic module from
http://rootiewiki.de/wiki/Serverstats (site seems down atm). To update
the stats, a crontab is run every minute:

* * * * * php5-cgi /var/www/feh.name/serverstats/update.php >/dev/null 2>&1

This works great... except that occasionally the Server load goes up
to something way beyond 20 (the last two times it was 25 and 34).
This has happened about five times in the last few months.
Then, the Cron daemon sends out a mail with the following content:

    /bin/sh: line 1: 21693 Bus error               php5-cgi \
    /var/www/feh.name/serverstats/update.php >/dev/null 2>&1

At the precise time this happens the traffic graph shows some sort of
"fluctuation", both in the RX and TX part.

The traffic source uses the /proc/net/dev file to calculate the
current traffic. Another tool that does that is `vnstat'. This tool,
however, doesn't show the `networking peak'.

Here you'll find two graphs. At approx. 8pm (20:00) the error happened:
http://feh.name/error_1.png
http://feh.name/error_1.png

I don't care if some source produces an error. But taking the load up
to more than 20 is not funny any more. :-/

Have you guys any idea?

I would appreciate every hint. Thank you.
