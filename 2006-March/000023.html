>> Reading up on RRDTOOL and the different way that it interpolates  
>> data.
>> What is a good launch frequency?  Every 5, 10, 15 mins?
>
> I use a frequency of one minute, but 5 minutes should be fine for most
> setups.
> But thats mainly a question about how accurate you want the graphs  
> to be.
>
>> Also is there a way to set the output to tell browsers not to  
>> cache the
>> graphs.  As for a while I believed that it was not working until a
>> browser refresh (and a graph folder delete) caused the new graphs to
>> appear.
>
> Perhaps the graphs were really not updated by the script. graph.php
> starts rrdtoo, with --lazy. So rrdtool only writes a new graph if  
> there
> is something new to see on it. From the man-page:
>   [-z|--lazy]
>     Only generate the graph if the current graph is out of date or not
>     existent.
>
> You can change this in graph.php (line 247):
>   $rrdgraph->setLazy(false);
> (Sorry, no config-option so far)
>
> Disabling the caching at all should give the same result.
> ($config['usecache'] in config/graph.php)
