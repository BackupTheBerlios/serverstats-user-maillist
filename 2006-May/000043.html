> When Zend Optimizer is installed on CentOS 4.3 i386 - the program now
> fails with this error:

I had very bad experience with using some PHP-cache and bleeding edge
PHP-features and this seems to be a problem with Zend Optimizer, too.

> Notice: Undefined variable: simple in
> /var/www/html/serverstats/config/graph.php on line 36
> [...]
> Line 36 in /serverstats/config/ is:
> simpleconfig::graph($config, $GLOBALS['simple']);

$GLOBALS is a reference to the configuration-root. This class supports
array-access for autoloading config-parts. So the access to
$GLOBALS['simple'] should load the config in config/simple.php. I think
this optimizer doesn't know that it is possible to override $GLOBALS.
I'm not sure if there are some reasons not to override $GLOBALS, but I
thought "$GLOBALS" would be a adequate name for the root-configuration
inside config-files. If you have some good reason I will rename it, of
course. Perhaps the people from Zend Optimizer can tell you more?
As a temporary workaround you can replace every "$GLOBALS" in
includes/config.class.php and config/* with some other name.
